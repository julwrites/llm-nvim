### Development

- **Testing**: Run tests using `./test/run.sh`. This ensures that your changes don't break existing functionality.
- **New Features**: When adding a new feature, please add corresponding tests in the `test/spec` directory.

### Pull Requests

- **CI**: Before submitting a pull request, make sure that the CI checks pass. You can run the tests locally to check for any issues.
- **Documentation**: If you add a new feature, please update the `README.md` file with the relevant documentation.

### Style

- **Code Style**: This project uses the standard Lua style guide. Please make sure that your code adheres to this style.
- **Commit Messages**: Please use clear and descriptive commit messages. This will help other contributors understand the changes that you have made.
